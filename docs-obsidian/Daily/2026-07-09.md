### Sesión 4 — Efectos ambientales + SectoresSection orbital + fixes CSS

#### Bugs resueltos
| Bug | Síntoma | Fix |
|-----|---------|-----|
| BUG-005 | Scroll horizontal en móvil | `html { overflow-x: clip }` + `overflow-hidden` en WorkProcess |

| BUG-007 | Botón "Ver servicios" 100% de ancho en móvil | `items-start` en el flex container del Hero |

#### Nuevo componente — `components/ui/rain-effect.tsx`
- Gotas de lluvia con CSS puro (spans con `linear-gradient`)
- Sin SVG — más ligero y más fácil de personalizar
- `--rain-duration`: 1.8–3.6s por gota, delay 0–4s, opacidad 0.28–0.76
- Ángulo ~10° (140px horizontal / 760px vertical)
- Usado en `BannerSection` con `count={50}`

#### Reescritura completa — `components/sections/SectoresSection.tsx`

**Primer intento (CSS doble-rotación):** FALLÓ. CSS `animation` que toca `transform` sobreescribe completamente el `transform` inline del mismo elemento. Las tarjetas ignoraban su posición orbital.

**Implementación final (Framer Motion):**
- `useMotionValue(angle)` compartido en el padre, avanzado con `useAnimationFrame`
- `OrbitCard` sub-componente con `useTransform` propio por tarjeta (cos/sin para x, y)
- `radius = size * 0.50` → efecto "desbordante": tarjetas salen del viewport en móvil
- Sin `overflow-hidden` en la sección → el `html { overflow-x: clip }` global maneja el clip
- Hover pausa la animación
- Tarjetas con foto Unsplash, gradient oscuro, icono badge, nombre del sector
- Título "Campo, finca y agroindustria" movido ARRIBA del div de órbita
- `mb-20` de separación título-órbita para que las tarjetas que desbordan hacia arriba no lo pisen
